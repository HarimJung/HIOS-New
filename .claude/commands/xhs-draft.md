# /xhs-draft — 샤오홍슈 포스트 초안 생성

주어진 주제를 샤오홍슈(小红书) 포스트로 변환합니다.

## 소스 확인
1. `01-Projects/Visual-Climate/06-Content-Engine/sources/` 에서 오늘 날짜의 digest 파일이 있으면 읽기
2. 없으면 주어진 주제만으로 작성

## 포스트 작성 규칙

### 언어
- **중국어 간체(简体中文)** 로 작성
- 절대 한국어/영어를 그대로 번역하지 않음
- 중국 MZ세대가 쓰는 자연스러운 톤
- 이모지 자연스럽게 사용 (과하지 않게, 3-5개)

### 구조 (小红书 알고리즘 최적화)
```
[제목] — 클릭을 유도하는 짧은 제목 (20자 이내)
[커버 텍스트] — 이미지에 들어갈 핵심 문구 (10자 이내)

[본문]
- 첫 줄: 후크 (질문 또는 놀라운 사실)
- 2-3줄: 핵심 인사이트 (데이터 포함)
- 리스트 형태의 팁 3-5개
- 마지막: CTA (저장/팔로우 유도)

[태그] #투자 #AI #理财 ... (8-12개)
```

### 카테고리별 톤
- **AI 관련:** 实用干货 (실용적 꿀팁) 느낌. "这个AI工具太好用了"
- **투자 관련:** 理性分析 (이성적 분석) 느낌. "一文看懂..." 절대 투자 조언 X, 정보 공유 O
- **데이터 시각화:** 教程 (튜토리얼) 느낌. "手把手教你..."

### 금융 콘텐츠 주의사항
- "투자 조언"이 아닌 "정보 공유/학습" 프레이밍
- "投资有风险" 문구 필수
- 구체적 종목 추천 절대 금지
- 수익률 과장 금지
- 트렌드/개념 설명에 집중

## 출력 형식
각 포스트를 아래 형식의 .md 파일로 생성:

```markdown
---
platform: xiaohongshu
date: YYYY-MM-DD
topic: [주제]
source: [reddit/x/original]
status: draft
tags: [xhs, 투자, AI]
---

# [제목]

## 커버 텍스트
[이미지에 들어갈 짧은 문구]

## 본문
[중국어 본문]

## 태그
#tag1 #tag2 ...

## 이미지 구성 제안
- 슬라이드 1: [내용]
- 슬라이드 2: [내용]
...
```

## 저장 위치
`01-Projects/Visual-Climate/02-Xiaohongshu/drafts/YYYY-MM-DD-{slug}.md`
