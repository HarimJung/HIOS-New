## Slide 6: Meeting Automation

**Before:** Record meeting → manually write notes → email action items
→ update spreadsheet → update calendar. Time: 45 minutes/meeting.

**After:** Drop transcript → AI generates notes, action items, calendar
events, stakeholder updates. Time: 2 minutes/meeting.

**How:**
- Granola records and transcribes
- `/meeting` command auto-processes
- Notes filed by project, action items tracked, calendar updated
- Stakeholder profiles enriched automatically

---

## Slide 7: Multi-Project Dashboard

**Before:** 4 separate folder systems, 4 status emails, 4 mental models.
Context switching costs 23 minutes per switch (UC Irvine research).

**After:** One daily brief. All projects. Prioritized by deadline and
client importance.

**How:**
- `/today` generates morning brief with all project statuses
- Smart priority engine considers deadlines, client tier, dependencies
- Weekly review auto-generated from daily activity
- Inbox zero: everything classified and routed

## Slide 8: Content Compounding

**The consulting-content flywheel:**

```
Consulting work (e.g., create infographic)
    → YouTube: "How I made this UN infographic"
    → Xiaohongshu: 3 design tips (in Chinese)
    → LinkedIn: "Data visualization for CSE programs"
    → Newsletter: Monthly "What I Shipped"
    → New consulting leads from content audience
    → More consulting work
```

One command: `/content "topic" --multi-platform`
generates all platform versions simultaneously.
